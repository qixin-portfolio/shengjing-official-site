## 1. 晟景小监工定位

晟景小监工是晟景装饰为“透明工地”服务设计的品牌 IP，用来表达施工过程可记录、关键节点可查看、业主沟通更清楚的服务理念。

对外统一称呼：

- 晟景小监工

核心角色：

- 认真负责的小工地管家
- 帮业主看进度、看照片、看节点、提醒验收
- 用人格化方式解释施工日报、现场照片、老板审核、业主查看等透明工地流程

不要写成：

- 真人宝宝
- 老板孩子
- 真实儿童形象
- 实际施工人员
- 真实客户或业主形象

推荐公开表达：

> 晟景小监工是晟景装饰透明工地服务的品牌 IP 形象，用来帮助业主理解施工日报、现场照片、节点记录、老板审核和业主查看等流程。

## 2. 素材分类

### A. 品牌 IP 形象图

用途：

- 品牌表达
- 功能解释
- 空状态
- 提示卡
- 页面中的轻量说明模块

要求：

- 优先使用 PNG 或 WebP
- 透明背景优先
- 尺寸建议不低于 1200px
- 不携带未经确认的电话、地址、二维码、年限、获奖、服务户数等文字

适合位置：

- `/about/` 中的小模块：“晟景小监工：透明工地服务形象”
- `/transparent-site/` 中的流程解释模块
- 后续透明工地功能说明卡片

### B. 透明工地功能图

用途：

- 解释施工日报
- 解释现场照片
- 解释老板审核
- 解释业主查看流程
- 解释设计确认和节点留痕

要求：

- 可以是品牌插画或功能示意图
- 必须明确标注为“功能示意”或“流程示意”
- 不能标注为真实小程序截图
- 不能标注为真实客户案例

推荐主题：

- 晟景小监工拿施工清单
- 晟景小监工拿手机展示透明工地
- 晟景小监工在工地现场检查
- 状态贴纸：已上传、已审核、去看看、施工有记录

### C. 透明工地实机截图

用途：

- 产品功能证据
- 展示小程序真实界面
- 支撑透明工地能力说明

要求：

- 必须来自真实小程序 UI
- 截图中不得出现客户姓名、电话、完整门牌、完整报价、未经确认二维码
- 如截图内容为演示账号或演示项目，必须标注“演示截图”
- 不得暗示已经大规模上线运营，除非有明确事实依据

建议收集截图：

- 小程序首页
- 工地列表
- 施工日报
- 施工进度
- 现场照片
- 设计方案
- 老板审核后展示页
- 业主查看页

### D. 真实工地现场照片

用途：

- 施工证据
- 工艺节点说明
- 案例补充
- 透明工地记录证明

要求：

- 必须是真实项目照片，不能用 AI 图替代
- 每张照片必须确认可公开展示
- 不出现客户姓名、电话、完整门牌、身份证、车牌等隐私信息
- 不把施工中照片写成完工实拍
- 不把效果图写成完工实拍

建议阶段：

- 防水阶段

- 安装阶段
- 完工阶段
