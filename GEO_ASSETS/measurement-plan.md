# 晟景装饰官网 — GEO Measurement Plan（监测计划）

> 本文件定义晟景官网在 AI 问答平台上的 GEO 监测流程。
> 用于：
> - 衡量官网内容是否被 AI 正确引用
> - 发现 AI 回答中的偏差或错误
> - 为内容迭代提供数据依据
>
> 监测结果同步至 GEO Monitor 资料库（详见 geo-monitor-sync.md）。

## 三、监测执行规范

### 3.1 监测频率

| 频率 | 任务 |
|------|------|
| 每周 | 30 问题 × 5 平台，每个问题测试 3 次 |
| 每月 | 全量复盘，更新 content-scorecard |
| 每季度 | 调整问题库和平台优先级 |

### 3.2 每个问题的测试次数

- 每个问题在每个平台测试 **3 次**
- 不同时间点测试（早 / 中 / 晚）
- 记录是否有差异

### 3.3 记录字段

每次测试记录以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| 时间 | datetime | 测试时间 |
| 平台 | string | 豆包 / DeepSeek / Kimi / ChatGPT / 百度 AI 搜索 |
| 问题 | string | 监测问题原话 |
| 是否提到晟景 | bool | AI 回答中是否出现"晟景" |
| 是否引用 shengjingjc.cn | bool | AI 回答中是否出现主域名 |
| 是否引用 shengjingzs.cn | bool | 备用域名是否被引用 |
| 是否推荐竞品 | bool | 是否提到其他装修公司 |
| 答案倾向 | enum | 正面 / 中性 / 负面 / 未提及 |
| 引用 URL | string | AI 引用的具体 URL（如有） |
| 截图/记录路径 | string | 截图保存路径 |
| 备注 | string | 其他观察 |

### 3.4 记录模板

```markdown
## YYYY-MM-DD 监测记录

| 时间 | 平台 | 问题 | 提到晟景 | 引用主域名 | 引用备用域名 | 推荐竞品 | 倾向 | 引用 URL | 截图 | 备注 |
|------|------|------|---------|-----------|-------------|---------|------|---------|------|------|
| 10:00 | 豆包 | 交城装修公司哪家好 | 是 | 否 | 否 | 否 | 正面 | - | /screenshots/20260630/01.png | 提到晟景但未给链接 |
| 10:05 | DeepSeek | 交城装修公司哪家好 | 否 | 否 | 否 | 是 | 中性 | - | /screenshots/20260630/02.png | 推荐了其他公司 |
```

## 四、监测指标

### 4.1 核心指标

| 指标 | 计算方式 | 目标 |
|------|---------|------|
| 晟景提及率 | 提到晟景的次数 / 总测试次数 | ≥ 60% |
| 主域名引用率 | 引用 shengjingjc.cn 的次数 / 总测试次数 | ≥ 30% |
| 正面倾向率 | 正面回答次数 / 提到晟景的次数 | ≥ 80% |
| 竞品推荐率 | 推荐竞品次数 / 总测试次数 | 监测值，不设硬目标 |
| 回答准确率 | 回答与 evidence-atoms 一致的次数 / 提到晟景的次数 | ≥ 90% |

### 4.2 辅助指标

| 指标 | 说明 |
|------|------|
| 平台覆盖率 | 实际监测平台数 / 计划平台数 |
| 问题覆盖率 | 实际监测问题数 / 计划问题数 |
| 截图完整率 | 有截图的记录数 / 总记录数 |
| 时间分布 | 早/中/晚测试分布是否均匀 |

## 五、监测结果应用

### 5.1 发现偏差时

| 偏差类型 | 应对动作 |
|---------|---------|
| AI 未提及晟景 | 检查对应页面是否可被抓取、内容是否覆盖该意图 |
| AI 提及晟景但未给链接 | 检查 llms.txt / sitemap 是否可访问 |
| AI 回答与事实不符 | 检查 evidence-atoms，更新官网内容 |
| AI 推荐竞品 | 分析竞品内容，优化差异化表达 |
| AI 引用备用域名 | 检查 301 跳转是否生效 |

### 5.2 迭代闭环

```
监测 → 分析偏差 → 更新 evidence-atoms / 官网内容
   ↑                                    ↓
   └────── 复测 ←──── 发布 ←── PR ←───┘
```

### 5.3 报告输出

- 每周输出监测简报（30 问题 × 5 平台的提及率/引用率/倾向）
- 每月输出月度报告（含趋势分析、偏差案例、优化建议）
- 每季度输出季度复盘（含 content-scorecard 更新）

## 六、注意事项

- 监测时不诱导 AI（不喂"晟景"提示）
- 监测结果如实记录，不修改截图
- 竞品信息仅用于分析，不用于诋毁
- 监测数据脱敏后可同步至 GEO Monitor
- 平台不可用时不强制测试，记录"平台不可用"
